## 시퀀스 구성

### Seq 1: seq1.tsx (0ms ~ 14800ms, 888 frames)
- **비주얼 컨셉 기획**:
  - 질문형 오프닝. 거대한 물음표 혹은 기어 아이콘 등장.
  - 화면 중앙에 "3-STEP" 이라는 키워드 등장(`Scale Punch`).
  - 우측 하단에서 `BG_SURFACE`를 적용한 단계별 패널의 밑바탕 구조가 윤곽선으로 빌드오버 됨.
- **세부 시퀀스 분할 계획**:
  - SubSeq 1 (0 ~ 280 frames): 화면 중앙에 질문의 뉘앙스를 주는 점멸하는 텍스트 블록 "HOW?" 렌더 (`Glitch Flash`).
  - SubSeq 2 (280 ~ 888 frames): "3-STEP WORKFLOW" 거대 자막 슬램 인. 백그라운드에 설계도면 모양의 격자 패턴 무빙.
- **전환 계획**: `Smash Cut`으로 격자무늬가 확 튀어나오며 장면 전환.

### Seq 2: seq2.tsx (14800ms ~ 26080ms, 677 frames)
- **비주얼 컨셉 기획**:
  - 1, 2, 3 단계가 스텝별로 순차적으로 불이 들어오는 트리 뷰 애니메이션.
  - 단계별로 `PRIMARY_GLOW` 색상 파티클이 각 층에 전송되는 연출. `STAGGER.LOOSE` 적용.
- **세부 시퀀스 분할 계획**:
  - SubSeq 1 (0 ~ 196 frames): STEP 1. Architecture 블록 활성화. 청사진(Blue Print) 실루엣 등장.
  - SubSeq 2 (196 ~ 436 frames): STEP 2. AI Draft 블록 활성화. 코드가 쏟아지는 빠른 스캔 이펙트 발생.
  - SubSeq 3 (436 ~ 677 frames): STEP 3. Refactoring 블록 활성화. 무질서하던 코드가 깔끔한 블록으로 정렬되며 핏이 맞춰지는 애니메이션 (`SPRINGS.SNAPPY`).
- **전환 계획**: 정렬된 블록 중 하나가 줌-인 되면서 전체 캔버스를 덮음. (Zoom In)

### Seq 3: seq3.tsx (26080ms ~ 38840ms, 766 frames)
- **비주얼 컨셉 기획**:
  - 단순 마크업 코드와 복잡한 비즈니스 로직의 코드가 빠르게 교차되며 나타남.
  - 좌측에 프롬프트 입력창을 연상시키는 모형 배치. 짧은 프롬프트 입력 직후 광선(Glow)이 번지며 Next.js 로고와 마름모형 컴포넌트 다이어그램이 완성됨.
- **세부 시퀀스 분할 계획**:
  - SubSeq 1 (0 ~ 233 frames): HTML 뼈대 모형이 나타났다가 `BG_DEEP` 안으로 페이드 아웃 되며 더 굳건한 컴포넌트 모형이 등장.
  - SubSeq 2 (233 ~ 766 frames): 프롬프트 터미널형 패널에 "Create Next.js Component..." 타이핑 직후 화려한 트리 다이어그램 확장(`Slam In`). API 연동, 상태관리가 라벨 바 형태로 팝업(`STAGGER.TIGHT`).
- **전환 계획**: Glitch Flash를 동반한 Smash Cut.
