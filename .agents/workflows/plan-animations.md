# /plan-animations {project_id}

scaffold가 완료된 프로젝트를 바탕으로, 각 섹션(예: intro, body1 ... outro)들의 원본 대본과 타임스탬프를 종합하여 최상의 결과물이 담길 애니메이션 "기획서"를 체계화합니다.

## 사전 조건

- 해당 프로젝트 `public/{project_id}/` 경로 내부에 각각의 섹션 파일 트리 구조가 정상적으로 생성되어 있어야 합니다 (`.txt`, `.wav`, `_timestamp.json`, `_context.md`)

## 워크플로우 단계

### 1. Remotion Best Practices 스킬 숙지

필요하다면 로컬 파일 시스템 리딩 도구를 이용해 `.agents/skills/remotion-best-practices/SKILL.md`를 먼저 읽고 Remotion 기본 패턴과 지식을 충전하세요.

### 2. 프로젝트 구조 (분석 대상) 동적 파악

디렉토리 확인 도구(`list_dir` 등)를 이용해 `public/{project_id}/` 디렉토리 하위에 실제로 존재하는 모든 하위 폴더(섹션) 리스트를 수집합니다.
(섹션 이름이 반드시 intro, body로 되어있지 않을 수 있으니 실제 탐색된 폴더들을 기반으로 처리해주세요.)

### 3. 디자인 시스템 (존재할 경우) 숙지

프로젝트에 적용시켜야 하는 고정된 디자인 템플릿 제약이 있다면 확인합니다.
보통 `public/{project_id}/design-system.md` 경로에 존재할 수 있습니다. 해당 문서가 없다면 패스합니다.

### 4. 각 섹션별 기획서 순차 작성

탐색 된 각 섹션 폴더를 하나씩 돌면서 다음 단계를 수행하세요.

#### 4-1. 컨텍스트 수집

`public/{project_id}/{section}/{section}_context.md` 파일을 파싱합니다.
여기에 적힌 원본 대본 텍스트와 하단의 타임스탬프 매핑 정보를 대조하며 분석하세요.

#### 4-2. 데이터 병합 보정 가이드

> ⚠️ [주의] 타임스탬프 객체의 텍스트는 Whisper AI 음성인식 엔진이 추출하여 원문과 오차가 나거나, 심지어 환각(없는 말 지어내기)이 존재할 확률이 높습니다!
>
> - **시퀀스 분할의 기준은 100% "원본 대본"을 따라야 합니다.**
> - 타임스탬프에서 얻어야 할 것은 글자 정보가 아니라 오직 타이밍(startMs / endMs) 값 뿐입니다.
> - Whisper가 마음대로 삽입한 잡음("감사합니다", "뉴스입니다" 등) 구간은 시퀀스 기획에서 과감히 제외하거나 무시하세요.
> - 여러 타임스탬프 블록이 하나의 대본 문장을 이루는 경우 묶어야 합니다.

#### 4-3. 시퀀스(seq) 분할 로직

긴 대본 흐름을 자연스럽고 몰입감 있게 시퀀스(`.tsx` 파일)들로 나눕니다.

- 보통 대본의 1~3개의 연관된 문장 흐름을 하나의 시퀀스로 할당합니다.
- 화면이 멈춰있어 지루하지 않게 최대한 애니메이션 호흡을 가져갈 수 있게 많은 단위로 자르는 것을 고려하되, 트랜지션이 너무 잦아 어지럽지 않은 균형점을 찾으세요.
- 할당된 각 시퀀스의 `durationInFrames` 는 `Math.ceil((endMs - startMs) / 1000 * 30)` 로 임시 지정하여 기획서에 노출합니다.

#### 4-4. 기획안(Plan) 문서 작성

템플릿 가이드에 맞춰 `public/{project_id}/{section}/{section}_plan.md` 파일을 생성하고 내용을 적어주세요.

**기획서 출력 템플릿:**

```markdown
# {Section} 애니메이션 기획서

## 섹션 개요

- 총 길이: {duration}ms ({frames} frames @30fps)
- 원본 대본 문장 수: N
- 시퀀스 수: M

## 시퀀스 구성

### Seq 1: seq1.tsx ({startMs}ms ~ {endMs}ms, {frames} frames)

- **원본 대본**: "추출된 실제 타겟 기준 원본 문장"
- **타이밍 근거**: 타임스탬프 #X ~ #Y 구간 기반 추정 (startMs ~ endMs)
- **비주얼 컨셉 기획**: (레이아웃 및 등장 요소)
  - **언어 정책**: 화면에 표시할 텍스트(UI)는 고유명사나 특정 약어를 제외하고 대본과 같은 언어로 번역하세요.
  - 배경:
  - 메인 요소:
  - 보조 엑스트라:
- **애니메이션 효과**:
  - In (등장):
  - In-Progress (진행 유지):
  - Out (퇴장):
- **캡션(Subtitle)**: 약 30자가 넘어갈 경우, 의미 단위에서 `\n` 등을 사용해 2줄로 분리하여 작성하되, 대본 원문은 무조건 100% 일치시킵니다.

### Seq 2: seq2.tsx ({startMs}ms ~ {endMs}ms, {frames} frames)

... (이하 루프)

## 전환 (트랜지션) 계획

- Seq 1 → Seq 2 간 체인 넘김 전환 효과 (fade / slide 등)
- ...

## 의존성 리소스 (Assets)

- 모델 자신이 나중에 AI 생성해야 할 이미지 소스 정리 / 사용 에셋들 명시
```

### 5. 승인 (User Review)

전 섹션에 대해 기획(Plan) 문서 발행이 완료되면 사용자에게 승인을 구하세요.
사용자의 피드백을 반영해 기획서를 수정한 뒤 다음 스텝(Implement workflow)으로 넘어가야 합니다.
